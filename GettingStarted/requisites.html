Dowloading a single AMIDST module with dependencies

Here we explain how to download a single AMIDST module with its dependencies. This procedure is useful if we want to access to the AMIDST functionality from a non maven project.

First of all, check that maven is properly installed in your system:

$ mvn -v

If everything is right, the following output (or similar) will be generated:

Java version: 1.8.0_73, vendor: Oracle Corporation
Java home: /Library/Java/JavaVirtualMachines/jdk1.8.0_73.jdk/Contents/Home/jre
Default locale: en_US, platform encoding: UTF-8
OS name: "mac os x", version: "10.11.3", arch: "x86_64", family: "mac"
In the location you prefer, create a new folder called, for instance, copydep:

$ mkdir copydep

Create a new file called pom.xml with the content shown below. Replace [moduleName] and [version] with the module you aim to use and its version (e.g. standardmodels and 0.4.2). You can check the list of AMIDST modules and versions.


<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
<modelVersion>4.0.0</modelVersion>

    <groupId>com.yourproject</groupId>
    <artifactId>usingAmidst</artifactId>
    <version>1.0</version>
    
    
	<repositories>
	<!-- AMIDST repository in github -->
	<repository>
		<id>amidstRepo</id> <!-- local identifier, it can be anything -->
		<url>https://raw.github.com/amidst/toolbox/mvn-repo/</url>
	</repository>

	<!-- ... -->

	</repositories> 

    <dependencies>
        <!-- Load any of the modules from AMIDST Toolbox -->
        <dependency>
            <groupId>eu.amidst</groupId>
            <artifactId> [moduleName] </artifactId>
            <version> [version] </version>
            <scope>compile</scope>
        </dependency>

        <!-- ... -->
    </dependencies>

For initializing the process, run the following command:

$ mvn dependency:copy-dependencies -DoutputDirectory=./lib

If the process finishes correctly, a folder lib will be created containing the library of the module and all its dependencies. For example, we download the module standardmodels-0.4.2, this folder will contain the following libraries:


commons-cli-1.2.jar               guava-18.0.jar
commons-lang3-3.3.2.jar           hugin-8.3.jar
commons-math-2.1.jar              huginlink-0.4.2.jar
commons-math3-3.5.jar             slf4j-api-1.7.7.jar
core-0.4.2.jar                    standardmodels-0.4.2.jar
core-dynamic-0.4.2.jar

Now, for the functionality of this AMIDST module, simply copy the downloaded libraries into a folder in the classpath of your project.
